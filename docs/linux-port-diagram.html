3 — Blockers & Alternatives
B-1 — HIGH
titleBarStyle: 'hiddenInset' on Linux
Creates ~80 px dead zone + doubled title bar. macOS-specific API, not supported on Linux.
★ Recommended
Option B: frame:true, titleBarStyle:'default' on Linux + CSS display:none on #titlebar div. Zero custom button code, looks native per distro.
Aframe:false + custom HTML close/min/max buttons (most control, more code)
CtitleBarStyle:'hidden' only — hides text, keeps native drag, slight visual inconsistency
B-2 — HIGH
postinstall references xcrun (Xcode-only)
Error message on Linux: "run: export SDKROOT=$(xcrun …)". electron-rebuild itself works cross-platform but the fallback hint is macOS-specific.
★ Recommended
Option A: Replace electron-rebuild in postinstall with electron-builder install-app-deps — the officially recommended cross-platform rebuild approach.
BKeep electron-rebuild, fix only the echo message text to be platform-neutral
DJS postinstall script branching on process.platform (overkill for a comment-level fix)
B-3 — CRITICAL
No linux electron-builder target defined
package.json build config has only mac targets. Running electron-builder on Linux produces nothing usable.
★ Recommended
Option C: AppImage + deb targets — covers Ubuntu/Debian with apt-installable .deb and all other distros via portable AppImage. ~4 lines in package.json.
AAppImage only — fastest, zero distro-specific packaging, no install needed
DAppImage + deb + rpm — maximum distro coverage, requires rpm tool on CI runner
FFlatpak/Snap — best sandboxed distribution, 1-2 days extra setup
B-4 — MEDIUM
.workflow in global extraResources
OpenInPrateekTerm.workflow is listed in global extraResources so electron-builder tries to bundle it into Linux builds too, causing a copy error.
★ Recommended
Option A: Move the entry from global extraResources into mac.extraResources. Single 1-line config move.
CbeforeBuild hook that removes .workflow dir if process.platform !== 'darwin' (overengineered)
B-5 — HIGH
CI only runs on macos-14 — no Linux build job
GitHub Actions workflow has a single "Build arm64 DMG" job on macos-14. There is no job to build or test on Linux, so Linux artifacts are never produced in CI.
★ Recommended
Option A: Add a parallel build-linux job on ubuntu-latest (x64) + optional ubuntu-24.04-arm (arm64). ~30 lines of YAML. Upload AppImage + .deb as artifacts, attach to GitHub Release.
BMatrix strategy [macos-14, ubuntu-latest] — fewer YAML lines but harder to handle OS-specific env vars
DCross-compile Linux on macOS runner via Docker — Docker not available on macos-14 by default
ESelf-hosted Linux runner — free on GitHub, requires maintaining a machine
